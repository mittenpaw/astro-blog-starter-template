## Astroを選んだ理由

ブログを作るにあたって、いくつかの選択肢を検討しました。
WordPress、Hugo、Next.js、Gatsby……候補はたくさんありましたが、最終的にAstroを選びました。

理由はシンプルで、**「コンテンツ重視」**というコンセプトが自分のニーズにぴったりだったからです。
ブログは記事が主役。余計なJavaScriptは極力送らないというAstroの思想は、潔くて気持ちいいです。

## セットアップの手軽さ

公式のブログスターターテンプレートを使ったので、セットアップはほぼゼロから始める必要がありませんでした。

```bash
npm create astro@latest -- --template blog
```

このコマンド一発でブログの雛形ができあがります。MDXサポート、RSSフィード、サイトマップも最初から含まれていて、かなりお得感があります。

## コンポーネントの書き心地

Astroのコンポーネント（`.astro`ファイル）は、HTMLに近い書き方ができてとても直感的です。
フロントマター部分にTypeScriptを書いて、テンプレート部分にHTMLを書く——この構造がスッキリしていて好きです。

```astro
---
const greeting = "こんにちは";
---

<h1>{greeting}、世界！</h1>
```

## まとめ

Astroは「ブログを作りたい」という用途には非常に適していると感じました。
難しい設定なしにパフォーマンスの高いサイトが作れる点が特に気に入っています。

引き続きカスタマイズを加えながら、このブログを育てていきたいと思います。
