このブログはMarkdownおよびMDX形式で記事を書いています。
MDXを使うと、Markdownの中にAstroコンポーネントを埋め込むことができます。

## Markdownの基本

日常的な記事はMarkdown（`.md`）で書いています。
見出し、リスト、コードブロックなど、一般的なMarkdown記法がそのまま使えます。

```markdown
## 見出し2

- リスト項目1
- リスト項目2

`インラインコード`
```

## MDXが便利な場面

MDX（`.mdx`）は、Markdownの中にJSX/Astroコンポーネントを埋め込める拡張フォーマットです。

たとえば、カスタムリンクコンポーネントを使いたい場合：

import HeaderLink from "../../components/HeaderLink.astro";

<HeaderLink href="/about">プロフィールページへ</HeaderLink>

## コードブロックのシンタックスハイライト

Astroはシンタックスハイライトをビルトインでサポートしています。

```typescript
const greet = (name: string): string => {
  return `こんにちは、${name}さん！`;
};

console.log(greet("義雄"));
```

```python
def greet(name: str) -> str:
    return f"こんにちは、{name}さん！"

print(greet("義雄"))
```

## 引用

> アウトプットなき学びは、インプットの無駄遣いである。

こういった引用もきれいに表示されます。

## まとめ

MarkdownとMDXを使い分けることで、シンプルな記事から少しリッチな記事まで柔軟に書けます。
まずはMarkdownで書き始めて、必要に応じてMDXに切り替えるのが自然な流れかと思います。
